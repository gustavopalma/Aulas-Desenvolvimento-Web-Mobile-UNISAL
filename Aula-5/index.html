Persistência de Dados no Android

Persistência de Dados no Android

  • A persistência de dados consiste, basicamente, no salvamento de dados. Persistir dados significa salva-los e no Android isso pode ser feito de diversas formas, tendo como principais:
    • Arquivos: Podem ser arquivos binários ou arquivos de texto, são salvos na memória Flash do telefone, podem ser públicos ou privados
    • DataStore(SharedPreferences): O DataStore, é a versão moderna do SharedPreferences, que é uma forma rápida para armazenar preferências do usuário, ou mesmo dados com baixo critério de confidencialidade
    • Bancos de Dados: Todo dispositivo Android possui embarcado uma versão do Banco de Dados SQLite, um pequeno banco de dados relacional, que permite a persistência segura de dados, suportando operações complexas, como JOINS ou criação de VIEWS

Persistência de Dados no Android

  • Apesar ddo SQLite suportar tabelas com até 1GB de tamanho, e lidar bem com operações complexas, originalmente as aplicações Android não foram pensadas para demandar um grande armazenamento de dados local .
  • Em outras, palavras, podems criar aplicações que consigam gerenciar grandes volumes de dados e o banco de dados pode lidar om eles, contudo não devemos.
  • Isso se deve em grande parte, por uma série de limitações e possibilidade de perda para os daods, como por exemplo:.
    • Se os dados forem armazenados apenas localmente, o usuário pode por descuido limpar o armazenamento local do telefone simplesmente excluir todos os dados sem possibilidade de recuperação.
    • Caso o usuŕio migre para um telefone mais novo, os dados não serão transferidos.
  • Para evitar esses problemas, o desenvolvedor deve optar pela integração da sua aplicação com serviços de nuvem. Esse conteúdo será abordado posteriormente

Persistência de utilizando Bancos de Dados Locais - SQLite

  • O SQLite é um SGBD (Sistema Gerenciador de Bando de Dados) relacional, projetado para ambientes embarcados;
  • Ao contrário de SGBDs como o MySQL ou SQL Server, o SQLite não precisa de um servidor rodando, ele implementa a estratégia 'Self Contained File'
  • É um SGBD multiplataforma de códdigo aberto

Persistência de utilizando Bancos de Dados Locais - Android Room E Couroutines

  • O Android Room, é um pequeno ORM (Object-Relational Mapping) desenvolvido pela Google para Android
  • O uso do Room, é encorajado pois reduz a complexidade e a necessidade de escrever códigos repetitivos uma vez que ele pode automatizar a geração de códigos SQL e mesmo alguns códigos de manipulação de dados

  • Quando estamos desenvolvendo métodos e funções utilizando os recursos das Coroutines, uma palavra chave muda totalmente o fluxo de execução da aplicação
          
suspend fun salvarPessoa() {
    dao.insert(pessoa)
}
  • Uma função marcada com suspend pode ser pausada e retomada sem bloquear a thread;
  • Ela só pode ser chamada dentro de outra coroutine ou de uma função suspend.
